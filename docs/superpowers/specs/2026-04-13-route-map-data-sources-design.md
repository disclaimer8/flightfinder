# Route Map Data Sources — Design Spec

**Date:** 2026-04-13  
**Status:** Approved

## Problem

The current route map uses OpenFlights `routes.dat` (last updated 2017). Data is stale — airlines have launched, merged, and closed since then. Amadeus Flight Inspiration Search is being deprecated and only works for a handful of airports in the test environment.

## Goal

Show flight routes for any airport in the world with honest visual indication of data freshness, using only free data sources.

## Data Sources

### 1. OpenSky Network (primary — live traffic)

- **What it is:** Community ADS-B network tracking real aircraft in the sky
- **API endpoint:** `GET https://opensky-network.org/api/flights/departure?airport={ICAO4}&begin={unix}&end={unix}`
- **Data window:** last 30 days of departures from any airport
- **Rate limit:** 400 calls/day (free account with login); 100 calls/day without credentials
- **Credentials:** `OPENSKY_USERNAME` + `OPENSKY_PASSWORD` env vars; if absent, server falls back to anonymous (100/day limit)
- **Code format:** ICAO 4-letter codes — converted to IATA via `airports.dat` (already has ICAO field)
- **Cache TTL:** 7 days per airport — avoids repeated calls, stays within rate limit
- **Coverage:** Excellent for Europe, North America, East Asia. Sparser in parts of Africa and Central Asia (fewer ground receivers)

### 2. Wikidata SPARQL (secondary — scheduled routes)

- **What it is:** Community-maintained knowledge graph with airline route timetables
- **Refresh:** GitHub Action runs every Monday at 03:00 UTC
- **Output:** `server/src/data/wikidata-routes.json` — Map of `src_iata → dst_iata[]`
- **Loaded:** At server startup; refreshed on every deploy
- **Coverage:** Complements OpenSky for routes with poor ADS-B coverage (remote regions, low-frequency routes not recently observed)
- **Rate limit:** None for SPARQL endpoint

### 3. OpenFlights routes.dat (fallback — hidden by default)

- **What it is:** Static dataset from 2017, 67k routes
- **Shown:** Only when user explicitly enables "Historical routes" toggle in map controls
- **Purpose:** Academic/exploratory use only — shows routes that may no longer operate

## Confidence Tiers

Each route gets assigned the **highest** confidence level from any source that knows about it. Sources are additive — a route can only be upgraded, never downgraded.

| Tier | Color | Condition |
|------|-------|-----------|
| **Live** | Green | OpenSky observed departure ≤ 7 days ago |
| **Recent** | Blue | OpenSky observed departure 8–30 days ago |
| **Scheduled** | Yellow | Wikidata only (not observed in OpenSky) |
| **Historical** | Gray dashed | routes.dat only — hidden by default |

When a route appears in multiple sources, it is assigned the highest-confidence tier. Example: MAD→JFK in both Wikidata and OpenSky (last 3 days) → shown as **Live** (green).

## Merge Algorithm

```
function mergeRoutes(iata):
  openSkyRoutes  = openSkyService.getDepartures(icao, 30 days)
  wikidataRoutes = wikidataService.getRoutes(iata)
  historicalRoutes = openFlights.getDirectDestinations(iata)

  allDestinations = union(openSkyRoutes.keys, wikidataRoutes, historicalRoutes)

  for each destination in allDestinations:
    if destination in openSkyRoutes:
      age = today - openSkyRoutes[destination].lastSeen
      confidence = age ≤ 7 ? 'live' : 'recent'
    else if destination in wikidataRoutes:
      confidence = 'scheduled'
    else:
      confidence = 'historical'

  return routes sorted by confidence desc
```

No conflict resolution needed — the hierarchy is unambiguous.

## Server Architecture

### New files

**`server/src/services/openSkyService.js`**
- `getDepartures(icao, daysBack)` → `{ dest_iata: string, lastSeen: Date }[]`
- Internal: fetches OpenSky API, converts ICAO destination codes to IATA, handles 404/empty gracefully
- Cache: in-memory Map with per-airport TTL (7 days)
- On rate limit (429): returns cached data if available, otherwise empty array

**`server/src/services/wikidataService.js`**
- Loads `wikidata-routes.json` at startup
- `getRoutes(iata)` → `Set<string>` of destination IATA codes
- No network calls at runtime — pure in-memory lookup

**`server/src/services/routesService.js`**
- Orchestrates all three sources
- `getRoutes(iata)` → `{ destinations: string[], confidences: Record<string, Tier>, prices: Record<string, number> }`
- Prices field kept for future use (currently empty without Amadeus)

### Modified files

**`server/src/controllers/mapController.js`** — `getRoutes` delegates to `routesService.js`

**`server/src/data/wikidata-routes.json`** — generated by weekly GitHub Action

### New GitHub Action

**`.github/workflows/wikidata-routes-refresh.yml`**
- Schedule: `cron: '0 3 * * 1'` (every Monday 03:00 UTC)
- Steps: checkout → node script to SPARQL query → write JSON → commit if changed → push

**`scripts/refresh-wikidata-routes.js`**
- SPARQL query to Wikidata endpoint (`https://query.wikidata.org/sparql`)
- Fetches airline route items (Wikidata class Q42240 = "flight route") using properties:
  - `wdt:P1825` = origin airport → `wdt:P238` = IATA code
  - `wdt:P1826` = destination airport → `wdt:P238` = IATA code
- Writes result to `server/src/data/wikidata-routes.json`
- Note: Wikidata route coverage varies by airline; large carriers (Ryanair, easyJet, Lufthansa, etc.) are well-represented; smaller regional carriers may be missing

## Client Changes

### RouteMap.jsx

- Arc color driven by confidence tier: green / blue / yellow / gray-dashed
- New state: `showHistorical` (default `false`)
- When `showHistorical` is false: filter out `confidence === 'historical'` before drawing arcs
- Map legend: small 4-row color key in bottom-right of map

### RouteMap.css

- `.rm-arc--live` — green, opacity 0.8
- `.rm-arc--recent` — blue, opacity 0.65
- `.rm-arc--scheduled` — yellow, opacity 0.5
- `.rm-arc--historical` — gray, dashed stroke, opacity 0.3

### Controls bar

New button: **"Historical routes"** toggle (off by default)  
Position: existing `.rm-controls` bar, after the radius slider

### Map legend

Small panel bottom-right, always visible when routes are loaded:
- 🟢 Live (< 7 days)
- 🔵 Recent (< 30 days)  
- 🟡 Scheduled
- *(Historical toggle only adds gray when enabled)*

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| OpenSky rate limited (429) | Serve cached data if available; else fall through to Wikidata + historical |
| OpenSky airport not found | Empty array — no error shown to user |
| Wikidata routes file missing | Log warning at startup; service returns empty set |
| ICAO code not found for IATA | Skip OpenSky query for that airport; Wikidata + historical still work |
| All sources return 0 routes | Show error: "No route data found for this airport" |

## What Is Explicitly Out of Scope

- Pricing data (removed with Amadeus)
- Real-time flight position tracking (OpenSky positions API)
- Cargo-only route filtering
- Codeshare deduplication
